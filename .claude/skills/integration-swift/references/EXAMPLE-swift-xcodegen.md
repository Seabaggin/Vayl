## README.md

# PostHog Swift + XcodeGen example

A minimal [SwiftUI](https://developer.apple.com/xcode/swiftui/) app whose Xcode
project is generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from
`project.yml`. It demonstrates the **correct** way to add PostHog to an
XcodeGen-managed project.

## The one rule that matters

In an XcodeGen project the `.xcodeproj` (including `project.pbxproj`) is a
**build artifact** — it is not committed, and `xcodegen generate` recreates it
from `project.yml` every time. Any dependency added by editing the pbxproj
directly (or through Xcode's *File > Add Package Dependencies* UI) is silently
wiped by the next generate.

So the PostHog SPM dependency lives in `project.yml`:

```yaml
packages:
  PostHog:
    url: https://github.com/PostHog/posthog-ios
    from: 3.64.1   # check github.com/PostHog/posthog-ios/releases for the latest

targets:
  BurritoStand:
    type: application
    platform: iOS
    sources: [BurritoStand]
    dependencies:
      - package: PostHog
```

How to tell a project is XcodeGen-managed: a `project.yml` at the repo root
with top-level `targets:` and/or `packages:` keys (the filename alone is not
enough — `project.yml` is a generic name), and typically no committed
`.xcodeproj`.

## Getting started

### 1. Generate the Xcode project

```bash
brew install xcodegen   # if needed
xcodegen generate
```

### 2. Set your PostHog project token

Open `BurritoStand/BurritoStandApp.swift` and replace the
`<your-project-token>` placeholder with your project token from your
[PostHog project settings](https://app.posthog.com/project/settings).

The PostHog project token is a **public client-side key** — it is designed to
ship in the app binary — so hardcoding it is safe and is the recommended
approach for iOS distribution.

### 3. Build and run

Open the generated `BurritoStand.xcodeproj` in Xcode and run on an iOS
Simulator. Xcode resolves the PostHog package automatically.

## Project structure

```
swift-xcodegen/
├── project.yml                    # XcodeGen spec — the source of truth,
│                                  # including the PostHog SPM dependency
└── BurritoStand/
    ├── BurritoStandApp.swift      # App entry point with PostHog initialization
    └── ContentView.swift          # Event capture and screen view tracking
```

## Key integration points

### Dependency declaration (project.yml)

The package is declared in the `packages:` block and referenced from the
target's `dependencies:` — never in the pbxproj. See the snippet above.

### PostHog initialization (BurritoStandApp.swift)

```swift
import PostHog

let config = PostHogConfig(apiKey: "<your-project-token>", host: "https://us.i.posthog.com")
config.captureApplicationLifecycleEvents = true
PostHogSDK.shared.setup(config)
```

### Event capture (ContentView.swift)

```swift
PostHogSDK.shared.capture("burrito_sold", properties: [
    "total_sold": burritosSold,
])
```

## BurritoStand/BurritoStandApp.swift

```swift
import SwiftUI
import PostHog

// PostHog configuration.
//
// The project token is a PUBLIC client-side key — it is designed to ship in the
// app binary, so hardcoding it here is safe and is the recommended approach for
// iOS. Replace the placeholder below with your project token from
// https://app.posthog.com/project/settings.
private let posthogProjectToken = "<your-project-token>"
private let posthogHost = "https://us.i.posthog.com"

@main
struct BurritoStandApp: App {
    init() {
        let config = PostHogConfig(apiKey: posthogProjectToken, host: posthogHost)
        config.captureApplicationLifecycleEvents = true
        config.debug = true
        PostHogSDK.shared.setup(config)
    }

    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}

```

## BurritoStand/ContentView.swift

```swift
import SwiftUI
import PostHog

struct ContentView: View {
    @State private var burritosSold = 0

    var body: some View {
        VStack(spacing: 16) {
            Text("🌯 Burrito Stand")
                .font(.largeTitle)
            Text("Burritos sold: \(burritosSold)")
                .font(.title2)
            Button("Sell a burrito") {
                burritosSold += 1
                PostHogSDK.shared.capture("burrito_sold", properties: [
                    "total_sold": burritosSold,
                ])
            }
            .buttonStyle(.borderedProminent)
        }
        .padding()
        .postHogScreenView("BurritoStand")
    }
}

#Preview {
    ContentView()
}

```

## project.yml

```yml
# XcodeGen project spec — the .xcodeproj is generated from this file and is
# NOT committed. Run `xcodegen generate` after editing, then open the project.
#
# This file is the single source of truth for the project structure, including
# the PostHog SPM dependency below. Never add dependencies by editing
# project.pbxproj directly in an XcodeGen project: the next `xcodegen generate`
# regenerates the .xcodeproj and silently wipes any direct edit.
name: BurritoStand
options:
  bundleIdPrefix: com.posthog.example
  deploymentTarget:
    iOS: "16.0"

# PostHog is declared here (not in Xcode's package UI). Check
# https://github.com/PostHog/posthog-ios/releases for the latest version.
packages:
  PostHog:
    url: https://github.com/PostHog/posthog-ios
    from: 3.64.1

targets:
  BurritoStand:
    type: application
    platform: iOS
    sources: [BurritoStand]
    dependencies:
      - package: PostHog
    info:
      path: BurritoStand/Info.plist
      properties:
        UILaunchScreen: {}

```
