The Pulse — alignment options

Three fixes for the same root cause, in increasing order of rigor. Everything else about the widget (wash, orb glow, hairline, humility-first copy) stays untouched — this is only about the vertical relationship between the orb and the text.

Diagnosis

Both halves of the card — the text stack (.right) and the orb+pill (.left) — are centered as independent groups against the whole card height, not against each other. In this mock, dormant (2-line hero + 1 sub-line) and active (1-line hero + 2 sub-lines) happen to add up to roughly the same total height, so it almost lines up.

But measure the actual gap from the orb's center to the hero line: dormant sits ~13px below the hero's last line, active sits ~27px below it. That's a real ~14px drift in the one relationship that matters — the orb reads as "the answer to this text" — and it'll be worse with real copy, and visible as a small jump every time a check-in flips the card from dormant to active.

V1 — Anchored HeroRecommended

Stop centering the text stack as a group. Top-anchor it (eyebrow always starts at the same offset) and give the hero line a reserved 2-line-tall slot with its text vertically centered inside that slot — so whether the hero wraps to 1 or 2 lines, its visual center lands at the same y every time. Pin the orb to that fixed offset instead of the card's overall center. Sub/today-lines free-flow below without moving anything above them.

V2 — Hero/Orb Row PairingAlternative

Instead of approximating the alignment with an offset, make it structurally impossible to drift: put the hero text and the orb in the same flex row with align-items:center, so they vertically-center against each other, guaranteed, regardless of line count. The eyebrow becomes its own full-width row above; sub/today-line become their own full-width row below. Neither can ever perturb the hero↔orb relationship because they're no longer in the same centering group.

Trade-off: more rewrite than V1 (new row structure). The orb's position relative to the whole card now shifts slightly between states (it sits a touch lower in dormant, since the hero row is taller there) — but its position relative to the hero, which is what your eye actually tracks, never drifts. This is the one I'd reach for if real Pulse copy turns out to vary more than these two examples.

V3 — Fixed Rhythm SlotsAlternative

Treat every line as a fixed-height row (eyebrow / hero / sub / today-line), each reserved regardless of whether it's populated, so the card's total height — and therefore the orb's centered position against it — is deterministic. This is the closest match to Vayl's token-driven layout philosophy (fixed row heights map directly to AppSpacing), and the safest against Dynamic Type or long real content.

Trade-off: the dormant card now carries a reserved blank row where the today-line would go — visible dead space (see the gap below "A quick check-in" above). That's the honest cost of pixel-perfect determinism, and it cuts against the widget's minimal, uncluttered feel. I'd only pick this if the empty slot reads as intentional breathing room once seen with real device chrome, not before.

Which one

VersionFixes the drift?EffortRisk
V1 Anchored HeroYes, for 1–2 line heroes (this mock's range)Smallest — 4 property changesHand-tuned offset could need revisiting for longer content
V2 Row PairingYes, structurally — any line countMedium — reshuffles the DOM/gridOrb's absolute position on the card shifts a little between states (fine, since they never render simultaneously)
V3 Fixed SlotsYes, perfectly — identical every timeMedium — grid rows + reserved empty rowDead space in dormant state; feels less organic

My call: ship V1. It's the smallest change from what's already mostly-working, and it fixes the actual complaint (orb drifting relative to the hero) without introducing V3's dead-space problem. Keep V2 in your pocket if real Pulse questions end up running longer/more variable than "How's your capacity?" — that's the point where V1's hand-tuned offset starts fighting you and V2's structural pairing earns its extra complexity.
