The Path — the trail, everywhere

One spatial metaphor, used consistently across every screen in the family: the path hub is a stack of mini-trails, node management happens over the trail, and the Support role sees a read-only version of the same trail rather than a different surface. Cohesive; the tradeoff is that utility screens (picking a path, reviewing as Support) carry the full visual weight of the trail even when they don't need it.

Where this set is strongest: total visual consistency — learn the trail once, recognise it everywhere, including the path hub and the Support view. Where it strains: the path hub's mini-trails and Support's read-only trail both carry serpentine-curve rendering for screens whose actual job is "pick one" and "review, don't act" — real geometry for jobs that may not need it.
