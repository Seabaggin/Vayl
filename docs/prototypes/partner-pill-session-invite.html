On the backend timing (your question): the invite doesn't fire on "Settle in" itself — Home's Settle-in only hands off to Play in release builds (onNavigateToPlay?(), no row created). The real trigger is PlayStore.builderDidFinish(_:), wired to the SessionBuilder's confirm action — the moment Partner B finishes selecting/trimming cards and commits to the plan. That's what calls realtime.openSession(...) and creates the lobby-status row this pill polls for via the same SessionEntryStore.fetchOpenSession mechanism already in production. For a Path quick-play card, there's no builder step, so the tap itself would be that same confirm moment.
