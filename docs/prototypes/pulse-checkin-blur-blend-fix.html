Proposed — blob coverage + label blend
Full field coverage, no deadzones at the corners/centre, no hard box or cross-seam — but each color still reads as its own zone, not one blended mush. Labels blend the same way the zones do, so they read as glow, not stamped text.

What maps to Swift

Replaces zoneBox (hard-clipped quarter-rectangle + corner-anchored RadialGradient) with a zoneBlob function: a circle sized field.size * 0.92, centred (not corner-anchored) at each quadrant's point, RadialGradient stops at 0% (full color) / 45% (half) / 85% (clear) — the exact recipe from pulse-teach-landing.html's .zoneBlob. The four are wrapped in .compositingGroup().blur(radius: size * 0.05).blendMode(.screen) — flatten, then blur (≈11px at this mock's 226px field, scaling to ≈13pt at the real 264-330pt field), then blend with the atmosphere. mix-blend-mode: screen on each label ≈ .blendMode(.screen) added to ghostLabel's Text. Zone colors/opacity values (0.20/0.16/0.32/0.23) are unchanged from the real code.
