Desire Map — starfield consistency + answer-pill redesign

Two gaps flagged: the ambient starfield only renders on the intro screen today (DesireMapView.swift gates starField to raterPhase == .start), so the rater and reveal screens go flat by comparison. And the answer pills (_RaterPill) are functionally fine but visually thin — one dot, one hint, one hardcoded selection color regardless of which answer you picked.

Recommendation: wire the existing dust-field (_bgStars) into the rating, reveal, and charted screens as-is — it's already built and this is really a wiring gap, not a new design. For the pills, Option A "Ignite" reads best in place: it fixes the per-answer color bug (selection now tints in that row's color, not always magenta/purple) and the spark-on-select echoes the star language the rest of the flow already speaks. Option C is the close second if you want more physical "card" weight; Option B's icon glyphs are the boldest but risk feeling busy against three other spectrum cues already on screen (dot, progress bar, category label).
